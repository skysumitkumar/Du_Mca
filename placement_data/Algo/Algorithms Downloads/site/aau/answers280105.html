Question 2 (sketch)

  • a) s1.make; s1.insert(3); s1.insert(2); s1.insert(5);
    s1.delete_min returns 2
  • b) remove in loop one minimal element from s1 and one from s2 and compare if they are the same; if both s1 and s2 become empty at the same time and all the removed numbers were the same, return TRUE, otherwise return FALSE
  • c) all are O(1) except for delete_min which is O(n)
  • d) first insert all elements from A to a temporary MIN_SET and then remove one by one elements from MIN_SET and insert them in this order into A. The elements in A has to be all different. Using singly-linked list the complexity is O(n2), using Fibonacci heaps it is O(n * log n).

Question 3 (sketch)

This is a minimum spanning tree problem.
  • a) yes, there is another solution with the same total cost
  • b) the problem is a graph problem with undirected edges, the question is to find a minimum spanning tree
  • c) e.g. by using Kruskal's algorithm, complexity is O(m * log n)
  • d) e.g. by modifying Kruskal's algorithm, first order the edges in such a way that they start with the links that are already built (and these edges are ordered) and then continues with the other edges (and these are again ordered). Do not include any edge which cannot be in the graph due to incompatibility. Finally run Kruskal's algorithm which processes the edges in this order.
