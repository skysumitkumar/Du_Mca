Datalight
Datalight makes Windows Embedded devices more reliable
Datalight products offer Microsoft Windows CE developers increased reliability, flexibility and confidence that their design will produce satisfied customers. Since 1983, Datalight's products have been integrated into: mobile phones, PDAs, GPS devices, data collection tools, industrial controllers, and navigation systems, allowing OEMs including Casio, Intel, and Symbol to save money, reduce development time and get to market faster. We pride ourselves in making products that solve real-world development issues.

The FlashFX Family includes single and multi-threaded flash drivers for devices using resident flash memory for mass storage. With support for more than 150 NOR and NAND flash parts, FlashFX and FlashFX Pro can be dropped into any Windows CE environment to increase product reliability or performance.

Datalight's Reliance file system prevents data corruption in the event of an unexpected power loss or system crash. It is the only file system designed specifically with the unique operating scenarios of embedded devices in mind.

Flash Driver Solutions

NAND Flash offers embedded developers obvious advantages. It is also, however, notoriously tricky to manage. Datalight’s FlashFX and FlashFX Pro offer developers a specific set of tools to make working with NAND easy.

Both FlashFX and FlashFX Pro offer rigorous bad block management, while delivering longer flash life through wear-leveling. FlashFX Pro is multi-threaded, and has been successfully implemented in consumer devices requiring the simultaneous support of multiple operations. Both products are ported to Windows CE.

Key benefits include:
  • Bad block management for NAND flash
  • Also supports NOR flash
  • Speeds time to market with built-in support for more than 150 different flash parts
  • Longer flash life lowers warranty costs

Reliable File System

Embedded devices need a file system that will not be corrupted when power is cut. Datalight's Reliance file system was written specifically for embedded devices and, unlike FAT-based file systems, ensures that both user and metadata are preserved in the event of an unexpected power down or system crash.

Reliance is a transactional file system that preserves the state of the disk by never overwriting good data. New data and file structures are always written to unused disk space, and do not become a part of the disk state until a transaction has completed. When a power glitch occurs, Reliance simply boots up and uses the last good committed state, ensuring that a Reliance disk is always consistent. Embedded devices that use Reliance are trustworthy and able to quickly boot.

Key Reliance benefits to the embedded developer include:
  • Highly reliable data storage
  • Faster boot because no CHKDSK utility is required
  • Full control over transaction settings to optimize performance
  • Faster and more efficient than Microsoft’s TFAT transactional alternative
  • Reliance is ported to Windows CE, speeding implementation
