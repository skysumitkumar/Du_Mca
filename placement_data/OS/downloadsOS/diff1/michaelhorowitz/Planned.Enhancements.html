Planned Enhancements to TimeKeeper

This page documents the changes and enhancements planned for the future - if I ever get to it. 
For changes already implemented, see the change history page 

Visible Changes 

  1. Add the ability to work on multiple projects 
  2. Make the text for the disabled buttons <small> so it is more obvious that this is not a valid option. This is mostly needed for Navigator v4 which does not support disabling of button.s 
  3. When suspend, include the length of the last interval in the main status box 
  4. Add a new cookie for TimeKeeper global options. First option is whether to display debugging information or not. 
  5. Nag mode: for suspensions of work that you expect to be short. Too often while using TimeKeeper I would suspend activity and forget to resume it when I started working again. Nag mode is intended for short work suspensions. With it, TimeKeeper will remind you (nag) that the clock is OFF every few minutes. Build this on top of the Timer Pop facility in the reminders. Maybe as a new type of reminder?
  6. Make the reminder window play a sound when the time is up. Can this be done and still keep TimeKeeper a single web page?
  7. Add an option to an expired reminder to expand it for another 5 minutes.  
  8. History functions: Add a Grand Total of time worked on a specific project across all days. Updated at end of day only. When reset? Save a log of every work interval? Save the time worked for the last few days? Save these things in a cookie and/or a file? 
  9. Let user chose to use the word "project" or "client" 
  10. For IE5 only, using DHTML, add a short display mode and a long display mode to hide all stuff except main buttons. Then again, adjusting the browser text size can get pretty close to this. 
  11. Put the input of project comments in a separate window. Maybe. 

Internal Changes (not visible) 

  1. Document why reminder alerts play a beep sometimes, but not always. 
  2. Make sure the Timekeeper section of the site is indexed by Atomz 
  3. Make a Style Sheet for the TimeKeeper section of computergripes.com 
  4. Convert reminders from using the onUnload event to the onBeforeunload event. I only generate this when running under IE anyway. Have to test for IE4 and above, this does not work with IE3 and below. Also change from warning alert to a question to give user a choice and not kill the reminder. Sometimes a web browser window is used when you don't intend it.  
  5. Control the exact placement of the help and reminder windows via x and y coordinates 
  6. Look up the default maximum comment size (its a text input field on a form)
  7. Use the PF1 key for help rather than a button. Work in Netscape? 
  8. For online version only: on page load go to the TopOfPage anchor to get rid of wasted space at the top of the page.
