int a ;

a=1 ;

printf("%d%d%d",a,++a,a++);

--------------------------

Here, it depends on intelligence of Compiler how it is going to generate code. I do not know compilers of others. But, VC++ compiler will generate code like following:

******
push 1
push 3
push 3
call printf
******

Which is perfectly valid though. like following interpretation you can do :

Step 1:

a = 1 ;
a++ ( Incremented but, not stored yet. Post increment )
so, push 1

Step 2:
Now, a is 2
++a ( Incremented and stored in that variable address. Pre increment. In case of pre increment after storing in that variable address then only it will proceed with further operations )
Now, a is 3
so, push 3

Step 3:
Now, a is 3
so, push 3

then call printf() function.

generally, they advise not to this kind of increment or decrement ( post of pre ) in the same statement. Since, it is confusion for programmer. But, at least as per calling convention of functions in C language, i guess answer should be proper and compiler should generate proper code. I do not understand if some compilers don't generate proper code though. After all, one compiler is generating expected answer why not others ? ( except bugs in compilers ) isn't it ?
