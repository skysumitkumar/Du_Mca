Issue with Settings.txt not saving the CabPoolPath location in Software Update Services, SUS

View products that this article applies to.
Author: Mohammed Athif Khaleel MVP

Last Review:June 28, 2005
Revision:1.0

SYMPTOMS

If you change the settings or options of Software Update Services, SUS from SUSADMIN page, sometimes, you notice, the changes are not saved. These changes are recorded in a file'C:\inetpub\wwwroot\autoupdate­\dictionaries\settings.txt'. For instance, you change the location of CabPoolPath and after reboot you see the changes are not saved.
Back to the top

CAUSE

The cause of this issue is the Software Update Services Synchronization Service (WUSyncSvc.exe) is running when you're trying to modify the file and hence, the file will be restored automatically. (WUSync Service enables Software Update Services to synchronize select content with the Microsoft Windows Update web site.)
Back to the top

RESOLUTION

To overcome such behavior and as a best practice, you have to stop Software Update Services Synchronization Service (WUSyncSvc.exe) before you change the settings and once the settings are changed, start the service. 

 
Back to the top

MORE INFORMATION

Follow these steps;
 
1. First, Stop SUS Synchronization Service; to stop the service, run this command from the command prompt;
Net stop WUSyncSvc.exe
 
2. Once the Service is stopped succesfully, modify the contents in 'C:\inetpub\wwwroot\autoupdate­\dictionaries\settings.txt' file & save it.
 
3. Start the SUS Synchronization Service; to start the service, run this command from the command prompt;
Net start WUSyncSvc.exe
 
4. Once the Service is started, make sure the changes are recorded successfully.
