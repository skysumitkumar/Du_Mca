    6. What is the difference between QC and QA?

      Quality assurance is the process where the documents for the proudct to be tested is verified with actual requirements of the customers. It includes inspection, auditing , code review , meeting etc.

      Quality control is the process where the product is actually executed and the expected behaviour is veritied by comparing with the actual behaviour of the software under test. All the testing types like black box testing, white box testing comes under quality control.

      Quality assurance is done before quality control.

      Tech Interviews comment by Pramod

    7. 5. How much interaction with users should testers have, and why?

      Here we have to understand , who are the users ? Are they really end users or the customer who is gives the testing job.
      Example, If a bank is given testing job to a “testing only” company, then, for the company, bank is the users. But actually, the users are who uses the software/product/service of the bank.

      If we are communicating with end users, then we may need to understand the requirements fully and start testing and also should suggest how the changes in developing can affect the user ? It is nothing but risk analysis.

      But is the user is the bank (in above example), then, some users expect daily testing status, daily progress report, test plan and test report along with other services for other user group (explained above).
      These are the deliverables the testing team should provide and the testing team should communicate with the bank not the end user directly. It is the bank’s job to understand and gather requirements from their end user (their customer)

    11. I am an IT Recruitment Manager and often have to interview testers. I find a useful question is to ask the candidate to describe a specific project and the testing lifecycle they went through. Did they write the master test plan? What did it include? Did they write test plans? What type of testing did they do? What did they base their test scripts on? How often did they receive a build to test? How did they create the test data?

      I also ask them:

      Tell me about a time when you received a late build to test, but the deadline for testing didn’t change. How did you meet the deadline?
      (I’m looking for a strategic approach, rather than just work harder/longer. ie. prioritising the test cases)

      Another good one is to ask:

      Tell me about a time when you received a build to test which differed to the original functional specifications. How did you deal with this? (hopefully this doesn’t happen too often, but I’m sure most testers have had to deal with this scenario. I’m looking for a pragmatic approach. Was the code just completely untestable, or were they able to create test scripts on the fly? If untestable, how did they communicate this back to the project manager?)

      Hope you find this useful.
      Cheers
